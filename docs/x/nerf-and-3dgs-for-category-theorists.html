NeRF & Gaussian Splatting

Sheaves that refuse to descend — and why the base object is optional

The Departure

Classical photogrammetry descends: it takes a cover (photographs), extracts a descent datum (feature correspondences), enforces the cocycle condition (bundle adjustment), and constructs the descended object (a mesh). The output is a concrete geometric object — vertices, edges, faces — living in \(\mathbb{R}^3\).

The methods on this page do something radical. They never construct the geometric object. Instead, they learn a function that, given a viewpoint, produces the image you would see. They encode the functor of views directly — the sheaf on the site — without ever materializing its representing object.

If photogrammetry is effective descent — recovering the object on the base from data on the cover — then NeRF and 3DGS are sheaves that refuse to descend. They stay at the level of the cover and say: the coherent family of views is all there is.

NeRF: The Implicit Sheaf

A Neural Radiance Field (Mildenhall et al., 2020) represents a scene as a single neural network — a multilayer perceptron (MLP) — that maps a 5D input to a 4D output:

\(F_\theta : (\underbrace{x, y, z}_{\text{position}},\ \underbrace{\theta, \phi}_{\text{view direction}}) \;\longmapsto\; (\underbrace{r, g, b}_{\text{color}},\ \underbrace{\sigma}_{\text{density}})\)

The network's weights \(\theta\) are optimized so that, for any camera pose, shooting rays through the scene and querying the MLP along each ray produces an image that matches the training photograph from that pose. There is no mesh. No point cloud. No triangles. The entire 3D scene is compressed into the weights of a neural network.

The 5D input is the key: the same spatial point can return different colors depending on the viewing direction. This is how NeRF captures view-dependent effects — specular reflections, glossy surfaces — that classical photogrammetry bakes into a single texture.

Ray Marching

To render a single pixel, NeRF casts a ray from the camera center through that pixel into the scene, then samples points along the ray at regular or stratified intervals. At each sample point, the MLP is queried to obtain color and density. This process — shooting a ray and stepping along it — is called ray marching.

Volume Rendering: Integrating the Ray

The color and density at each sample point are combined by the volume rendering equation:

\(\hat{C}(\mathbf{r}) = \sum_{i=1}^{N} T_i \cdot \alpha_i \cdot \mathbf{c}_i \qquad\text{where}\quad T_i = \prod_{j=1}^{i-1}(1-\alpha_j),\quad \alpha_i = 1-e^{-\sigma_i \delta_i}\)

Here \(T_i\) is the transmittance — the probability that the ray has not yet been absorbed before reaching sample \(i\). Each sample contributes its color weighted by how opaque it is (\(\alpha_i\)) and how much light reaches it (\(T_i\)). This is fully differentiable, so the whole pipeline — ray casting, MLP query, volume rendering — can be trained end-to-end by backpropagating the pixel-wise loss.

The key insight: volume rendering turns a continuous implicit function into a pixel color through integration. The scene has no surfaces — only a density field. Surfaces emerge as regions where \(\sigma\) is high. This is why NeRF can handle semi-transparent objects, fog, and fuzzy boundaries that defeat classical photogrammetry.

3D Gaussian Splatting: The Explicit Alternative

3D Gaussian Splatting (Kerbl et al., SIGGRAPH 2023) takes the opposite approach to NeRF's implicit representation. Instead of encoding the scene as a neural network, it represents it as a collection of millions of 3D Gaussian ellipsoids, each with explicit parameters:

Each Gaussian \(g_k\) carries: position \(\boldsymbol{\mu}_k \in \mathbb{R}^3\), covariance \(\Sigma_k \in \mathbb{R}^{3\times 3}\), color \(\mathbf{c}_k\) (via spherical harmonics for view-dependence), and opacity \(\alpha_k \in [0,1]\).

To render: project each Gaussian onto the image plane (producing a 2D Gaussian "splat"), sort by depth, and alpha-blend front-to-back. No neural network is queried at render time — just GPU rasterization. This is why 3DGS renders at 100+ FPS while NeRF struggles to reach 5.

Splatting: From 3D Blobs to Pixels

The rendering formula for 3DGS is remarkably similar to NeRF's volume rendering. For a pixel \(p\), sort the Gaussians by depth and blend:

\(\hat{C}(p) = \sum_{k \in \mathcal{N}} T_k \cdot \alpha_k \cdot G_k(p) \cdot \mathbf{c}_k \qquad\text{where}\quad T_k = \prod_{j=1}^{k-1}(1 - \alpha_j \cdot G_j(p))\)

Here \(G_k(p)\) is the 2D Gaussian evaluated at pixel \(p\) — how much the \(k\)-th splat covers this pixel. The formulas are nearly identical to NeRF's; the difference is computational: instead of marching a ray and querying a neural network at every sample, you project a fixed set of blobs and rasterize them with GPU hardware. The image formation model is the same. The implementation is radically different.

The Comparison

NeRF — Neural Radiance Field

implicit~5 FPS

Scene = weights of a neural network. Render by marching rays and querying MLP per sample. Captures view-dependent effects via directional input. Training: hours. Rendering: slow (network inference per pixel × samples per ray).

Categorically: the sheaf itself. Stores \(T \mapsto \mathrm{view}(T)\) as a learned function. Never materializes a base object.

3DGS — 3D Gaussian Splatting

explicit100+ FPS

Scene = millions of Gaussian ellipsoids with position, shape, color, opacity. Render by projecting and rasterizing splats. View-dependence via spherical harmonics. Training: minutes. Rendering: real-time (GPU rasterization, no neural network).

Categorically: a representing object — but made of probabilistic blobs rather than simplices. An explicit model you can manipulate.

Both share the same training signal (photographic images + camera poses from SfM) and the same image formation model (alpha-blending along depth-sorted contributions). They differ in representation: NeRF compresses everything into network weights (implicit), 3DGS stores explicit geometric primitives (millions of Gaussians).

The Categorical Reading

Here is the punchline, in the language you came for.

Classical photogrammetry performs effective descent. It recovers the object \(E\) on the base \(X\) from coherent local data on the cover. The output is a concrete geometric object — a mesh, a representing object for the functor of views.

NeRF encodes the sheaf itself. The MLP is the functor \(T \mapsto (\text{color}, \text{density})\) — it takes a probe (position + direction) and returns an observation. No representing object is ever constructed. If you want a mesh, you can extract one (via marching cubes on the density field), but this is an afterthought. The sheaf is primary. The space is a convenience.

3DGS constructs an explicit representing object — but a strange one. Instead of a simplicial complex (mesh), it produces a measure-theoretic one: a collection of Gaussian measures in \(\mathbb{R}^3\), each carrying color and opacity data. The "3D object" is a signed measure on \(\mathbb{R}^3\) with values in a color space. It is explicit and manipulable, but it is not a manifold. It is not even a CW-complex. It is a cloud of soft blobs.

The progression tells a story about what it means for a geometric object to "exist":

Photogrammetry: the object exists as a subspace of \(\mathbb{R}^3\) (a mesh). Descent is effective.
3DGS: the object exists as a measure on \(\mathbb{R}^3\). Descent produces something, but not a space — a distribution.
NeRF: the object does not exist at all. Only the functor of views exists. Descent is refused. The sheaf is the thing.

This is Grothendieck's insight arriving from engineering: you don't need the space. You need the site and the sheaf. The space, if it exists, is a representing object for the functor. But the functor is primary.
