The shape of a formula, read from the data

A neural network with a million weights memorizes. A formula with seven symbols explains. Between those two, AI Feynman 2.0 proposes a different kind of fitting: one that searches for the structure a formula would have if one existed — and then writes it down.

Given a cloud of points $\{(x_1^{(i)}, x_2^{(i)}, \ldots, x_n^{(i)}, y^{(i)})\}$, what equation $y = f(x_1,\ldots,x_n)$ produced them? This is symbolic regression: not fitting coefficients to a fixed template, but searching the combinatorial wilderness of all possible expressions built from $+, \times, \sin, \log, \exp, \ldots$ for the one that fits.

The space is vast — the number of expressions of length $L$ grows super-exponentially in $L$ — and yet every high-school physics textbook is full of short formulas that describe the world. The premise of the AI Feynman project is that this is not a coincidence. Short formulas exist because the world has structure, and that structure leaves fingerprints in the data. If we learn to read the fingerprints, we can narrow the search from all expressions down to a tiny, plausible few.

"We present an improved method for symbolic regression that seeks to fit data to formulas that are Pareto-optimal, in the sense of having the best accuracy for a given complexity." — Udrescu et al., 2020

This essay unpacks the central trick of the paper: the discovery that the gradient of a neural network fit — not its output, not its weights, but the geometry of its gradient vector field — reveals the modular structure of the underlying formula. From that structure, recursion handles the rest.

§ 2Pareto: accuracy against complexity

A straight line through the data has error $\varepsilon$ and complexity $c$ (say, measured in bits to describe the formula). A 15-term polynomial has lower $\varepsilon$ but much higher $c$. Neither is "the answer." The answer is the Pareto frontier: the set of formulas such that no other formula is both simpler and more accurate.

The old AI Feynman 1.0 returned a single best equation. The 2.0 version returns the frontier. This matters because real data is noisy: the "most accurate" formula often overfits by encoding the noise, and a slightly simpler formula on the frontier is frequently the true one. Handing the scientist the frontier and letting them choose is epistemically honest in a way that a single answer is not.

§ 3The gradient whispers its structure

Here is the heart of the paper. Suppose the true formula has the form

$f(x_1, x_2) = g(x_1 + x_2)$    (additive symmetry)

for some unknown $g$. Then by the chain rule:

$\displaystyle \frac{\partial f}{\partial x_1} = g'(x_1+x_2) = \frac{\partial f}{\partial x_2}.$

The gradient components are equal everywhere. If instead $f(x_1, x_2) = g(x_1 \cdot x_2)$, then $\partial f/\partial x_1 = x_2 \, g'$ and $\partial f/\partial x_2 = x_1 \, g'$, so their ratio is $x_2/x_1$ — a simple, checkable function of the inputs. Each kind of generalized symmetry — additive, multiplicative, ratios, rotational — leaves a distinctive fingerprint in the gradient field.

You don't know $f$, but you can fit a neural network $\hat f_\theta \approx f$ to the data and then ask it for its gradients via backpropagation. Modern autodiff turns this into one line of code. Picking a symmetry…
Fig. 2 · Left: the fitted function surface. Right: the gradient field $(\partial\hat f/\partial x_1, \partial\hat f/\partial x_2)$ colored by $\log(\partial_1/\partial_2)$. Constant color = symmetry detected.

That coloring is the whole trick. Where the ratio $\partial_1 \hat f / \partial_2 \hat f$ is constant, you have additive symmetry. Where it equals $x_2/x_1$, you have multiplicative symmetry. Where it equals $-x_2^2/x_1^2$ for ratio form, and so on. The paper formalizes this into a generalized symmetry detector: given any candidate functional form $h(x_1,x_2)$ such that $f = g(h)$, check whether $\nabla h$ is parallel to $\nabla \hat f$ pointwise. It's a small library of templates, each a few lines of linear algebra, each checkable in microseconds.

Once a symmetry is detected — say, $f(x_1,x_2,x_3) = g(x_1 + x_2, x_3)$ — the problem has been reduced. Two variables collapse into one, the fitting problem drops a dimension, and the symbolic search becomes tractable. Do it again on $g$. And again.

§ 4Recursive decomposition

A formula is not a flat string of symbols; it is a tree. And the tree has modularity: subtrees represent intermediate quantities. The ideal gas law $PV = nRT$ can be read as $h(P, V) = k(n, R, T)$, where $h$ and $k$ are each products. The gradient-symmetry test detects exactly this kind of split.

The solver's loop, in essence:

# Given samples of f: R^n -> R
def solve(samples):
    if n <= threshold:
        return brute_force_search(samples)    # small: just try formulas
    nn = fit_neural_net(samples)
    for template in [additive, multiplicative, ratio, ...]:
        if gradient_matches(nn, template):
            sub_samples = project_along(template, samples)
            return template.compose(solve(sub_samples))
    return dimensional_analysis_then_search(samples)

Each successful decomposition halves — sometimes quarters — the dimension of the remaining problem. The recursion bottoms out when only a few variables are left, at which point brute-force enumeration over short expressions (accelerated by dimensional analysis and statistical hypothesis testing on the candidates) finishes the job.

Each split is a symmetry discovery. The leaves are brute-force-findable.

§ 5Averaging, or: what noise reveals

One unexpected lesson in the original TDS kick-the-tires review: when the solver is given duplicate variables perturbed by independent noise, it learns — on its own — to average them. Given $x_0 = x_1 = a + \epsilon$ with independent Gaussian $\epsilon$, the symmetry detector finds $f(x_0, x_1) = g\!\left(\tfrac{x_0 + x_1}{2}\right)$, because that's the functional form minimizing MEDL on the Pareto frontier.

This is striking. A standard regression would treat $x_0$ and $x_1$ as two distinct noisy features. The symbolic solver identifies them as measurements of the same latent, and combines them in the variance-minimizing way. The drop in the signal-to-noise is visible below — drag the slider.

The same trick generalizes. If the solver posits $y = f(\bar x_{01}, \bar x_{23}) + \text{residual}$ and the residual becomes small enough to be classified as noise by a statistical hypothesis test, the averaged formula is accepted. This is where the paper's statistical hypothesis testing to accelerate brute-force search does real work: most candidate sub-formulas can be rejected on a handful of samples without a full fit.

§ 6What it means

There is a temptation to read AI Feynman 2.0 as a curiosity — a tool for rediscovering equations a physicist already knows. That reading underrates it. The method demonstrates something that was easy to doubt: that the geometry of a neural network's gradient field encodes interpretable, transferable structure. A fit that was previously treated as an opaque approximator turns out to be diagnostic: it tells you, via its derivatives, which variables the true function combines, and how.

That has implications beyond symbolic regression. Any pipeline that fits a neural network to data and then asks "what did it learn?" can, in principle, ask the same question of its gradients. The answer — a graph of modular dependencies — is a first-class object of mathematical interest. It is, after all, close to what a physicist means by a theory.

The neural network was always trying to tell us something. AI Feynman 2.0 is the method for listening at the right frequency.
