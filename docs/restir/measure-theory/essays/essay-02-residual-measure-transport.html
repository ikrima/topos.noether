Essay 2 · ReSTIR as Predictive Measure Dynamics

The renderer samples
what prediction failed to explain

A visual essay on residual-streaming ReSTIR: push a prior reservoir forward, compare it with incoming evidence, then spend samples on the residual measure.

For principal graphics researchers: this essay uses measure language only where it buys something concrete—reuse confidence, sample allocation, novelty detection, and curvature-aware rejection.

Blue: predicted measure from history. Gold: incoming evidence. Red: residual novelty that deserves fresh samples.
The key object is not the current frame alone. It is the difference between what the renderer predicted and what the frame now demands.

The usual rendering question is “where should I sample?” The residual version is sharper: which parts of the current measure are not already explained by transported history and neighbors?

ReSTIR already contains the ingredients of a predictive system. A reservoir at frame t−1 is transported forward through motion vectors, surface shifts, or GRIS maps. That transported reservoir is not merely old data. It is a prediction.

μt− = (Tt)# μt−1push the previous reservoir measure forward to predict the current one

Then the current frame provides evidence: direct candidates, visibility tests, shading probes, G-buffer changes, or whatever cheap signal we are willing to treat as an incoming measure.

ρt = νt − μt−the residual measure: current evidence minus predicted belief
This is the biomimetic ReSTIR principle: reservoirs should carry the stable predictable measure; new samples should target residual novelty.

1. Residual is a signed diagnostic, not just an error number

If incoming evidence is larger than prediction, the residual is positive: something new needs samples. If prediction is larger than evidence, the residual is negative: history is overconfident, stale, or invalid. Both are useful.

Path coordinate is deliberately abstract: light choice, direction, path shift parameter, or reservoir hypothesis index.
Positive residual means the current frame contains measure mass not predicted by the reservoir. Negative residual means the prior predicts mass the current frame no longer supports.

2. Expected-but-reweighted versus genuinely novel

The residual view becomes stronger when we separate two failure modes.

νt = ft μt− + νt⊥Lebesgue decomposition: reweighted expectation plus singular novelty

The first part, f μ⁻, says: “the old support was basically right, but weights changed.” The second part, ν⊥, says: “the current frame contains mass outside the predicted support.” That is disocclusion, new visibility, a light turning on, a caustic entering the pixel, or any event the transported reservoir cannot represent.

Absolutely continuous residual
Same candidate family, different density. Reweight, downweight history, or request a few corrective samples.
Singular residual
New support. History cannot explain it. Allocate fresh exploration and reduce reuse confidence sharply.
The dashed blue support is what history can explain. Red particles outside it are singular novelty.
A stable region can be handled mostly by density correction. A singular region requires exploration because no amount of reweighting creates samples where the prior has no support.

3. First-order residual is mismatch; second-order residual is alarm

A persistent error is bad, but a changing error is worse. It says the failure itself is moving, growing, or failing to transport. That is closer to an attention signal.

Δet = et − (St)# et−1second-order residual: where prediction error itself fails to predict

In a renderer, this distinguishes a stable under-sampled shadow from an emerging disocclusion, thin-geometry instability, or specular lobe that is moving faster than the reuse operator can track.

4. Attention is sample allocation over residual measure

Now the sample-budget field can be expressed as a sum of cheap diagnostics:

Ap = α |ρp| + β |Δρp| + γ νp⊥ + η κpallocate fresh work where prediction fails, changes, becomes singular, or transport does not commute

This is not “sample where bright.” It is “sample where the current evidence is poorly explained by transported belief.”

Rings show extra samples. Dark cells reuse aggressively. Red cells distrust history.
This is the operational version: convert residual diagnostics into sample budget and reuse confidence.

5. Curvature is residual disagreement between reuse paths

Temporal prediction and spatial prediction are both pushforwards. The interesting object is where they disagree.

κ = D((Ttime)#(Tspace)# μ, (Tspace)#(Ttime)# μ)if reuse order matters, the local sampling geometry is curved

That curvature term belongs in the attention field because it identifies places where the renderer’s transport model is locally inconsistent.

Two paths around the same screen-time square. When the final clouds separate, reuse is path-dependent.
High curvature should reduce reuse confidence even if each individual reprojection test looks plausible.

6. A renderer loop

The algorithm is not a replacement for ReSTIR. It is a control layer around it: a residual field that decides where to reuse, where to reweight, and where to explore.

for each pixel p:
    // 1. Predict from transported reservoirs.
    temporal = pushforward(prevReservoir[p'], motionOrPathShift)
    spatial  = pushforward(neighborReservoirs[p], localReuseMaps)
    predicted = combine(temporal, spatial)

    // 2. Obtain cheap current evidence.
    current = estimateIncomingMeasure(p)  // probes, candidates, visibility, G-buffer evidence

    // 3. Compare prediction with evidence.
    residual       = current - predicted
    residualChange = residual - pushforward(prevResidual[p'], residualTransport)
    novelty        = singularComponent(current, predicted.support)
    curvature      = distance(timeThenSpace(p), spaceThenTime(p))

    // 4. Convert residual diagnostics into policy.
    attention = a*norm(residual)
              + b*norm(residualChange)
              + c*novelty
              + d*curvature

    reuseConfidence = exp(-lambda * attention)
    sampleBudget    = baseSamples + allocate(attention)

    // 5. ReSTIR proceeds, but guided by residual measure.
    reservoir[p] = resample(predicted, freshSamples(sampleBudget), reuseConfidence)

7. The compressed formulation

Residual Measure Transport ReSTIR treats the transported reservoir as a predicted measure, compares it against incoming evidence, decomposes the mismatch into reweightable and singular components, and allocates fresh samples according to first-order residual, residual acceleration, singular novelty, and transport curvature.

In this picture, the reservoir is the stable stream. The residual is the learning signal. Reuse becomes confidence-weighted prediction; sampling becomes residual exploration.
