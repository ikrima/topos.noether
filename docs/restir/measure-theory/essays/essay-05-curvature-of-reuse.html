Curvature of Reuse

A renderer becomes unstable when two reasonable reuse paths produce different reservoirs.

Temporal reuse and spatial reuse are individually useful. Trouble begins when their order matters. In that small disagreement lives a practical signal for ghosting, shimmer, fireflies, stale shadows, disocclusions, and unsafe reservoir reuse.

The smallest loop in a renderer

Pick a pixel in the current frame. It can borrow from the previous frame, and it can borrow from a spatial neighbor. Those two operations form a little square in screen-time. If the world were perfectly flat in this transport sense, going around the square in either order would land in the same reservoir.

This is the graphics version of curvature: not a bent surface in space, but path dependence in reuse. A flat reuse field commutes. A curved reuse field remembers the route you took.

The commutator is a microscope

Curvature is often introduced as an abstract commutator. Here it has a mundane engineering meaning: apply two reuse operators in both orders and compare the resulting reservoirs.

[Ttime, Tspace]R = TtimeTspaceR − TspaceTtimeR

For an implementation, the minus sign does not have to be literal subtraction of samples. It can be any useful disagreement measure: luminance delta, visibility disagreement, entropy change, sample identity mismatch, Fisher-like weight divergence, Wasserstein-like centroid motion, or Jacobian instability.

Each tab shows a different cheap proxy for distributional disagreement between the two transported reservoirs.
Implementation bias: use the cheapest stable proxy first. A noisy curvature signal is worse than a simple one. Start with visibility mismatch, luminance variance, depth/normal derivative, and sample-ID disagreement; only then try distribution distances.

Where curvature shows up as artifacts

A high-curvature pixel is saying: “my transported past and borrowed neighborhood disagree about what should be here.” That is exactly the condition that produces many temporal reuse failures.

Flat reuse

Diffuse, stable, slowly moving regions. Temporal and spatial paths agree. Reuse should be aggressive.

Curved reuse

Disocclusions, moving shadow edges, thin glossy highlights, caustic folds. Reuse should be gated, refreshed, or split.

Curvature becomes a reuse valve

The curvature number is not useful because it sounds geometric. It is useful because it can directly modulate the two decisions every real-time renderer cares about: how much to trust history and where to spend new samples.

reuseTrust = exp(−λκ κ − λρ|ρ| − λn novelty)
Trust falls exponentially as curvature, residual, and novelty increase. Sample budget rises in the opposite direction.
float trust = exp(
    -lambdaK * curvature
    -lambdaR * residual
    -lambdaN * novelty);

reuseConfidence *= trust;
extraSamples = allocate(1.0 - trust);

A practical curvature pass

The literal commutator does not need to be expensive. For production, compute a proxy field that asks a cheaper question: “do my temporal and spatial predictions agree enough to reuse?”

motion vector mismatch visibility flip sample ID disagreement target PDF ratio Jacobian spread entropy delta luminance residual 
struct CurvatureInputs {
    Reservoir temporalPrediction;     // T_time(R_prev)
    Reservoir spatialPrediction;      // T_space(R_neighbors)
    Reservoir temporalThenSpatial;    // T_space(T_time(R_prev))
    Reservoir spatialThenTemporal;    // T_time(T_space(R_neighbors_prev))
    GBuffer   gbuffer;
    float     residual;
    float     novelty;
};

float estimateReuseCurvature(CurvatureInputs x)
{
    float idMismatch  = sampleIdDisagreement(
        x.temporalThenSpatial, x.spatialThenTemporal);

    float visMismatch = visibilityDisagreement(
        x.temporalThenSpatial, x.spatialThenTemporal, x.gbuffer);

    float pdfMismatch = abs(logSafe(
        targetPdf(x.temporalThenSpatial) /
        targetPdf(x.spatialThenTemporal)));

    float entropyGap  = abs(
        entropy(x.temporalThenSpatial) -
        entropy(x.spatialThenTemporal));

    return saturate(
        0.35 * visMismatch +
        0.25 * pdfMismatch +
        0.20 * idMismatch +
        0.20 * entropyGap);
}

void applyCurvatureGate(inout Reservoir r, float curvature, float residual, float novelty)
{
    float trust = exp(-2.2 * curvature - 1.4 * residual - 3.0 * novelty);
    r.confidence *= trust;
    r.extraSampleHint += sampleBudgetFrom(1.0 - trust);
}

The sentence to carry forward

Curvature is the local failure of reuse paths to agree. In a renderer, that failure is not philosophical. It is a measurable instability field that can reduce stale reuse, increase fresh sampling, allocate auxiliary reservoirs, and explain where temporal artifacts are born.
