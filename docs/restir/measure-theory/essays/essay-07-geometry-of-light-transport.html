The rendering equation is a test against a measure

For a pixel, there is a hidden set of possible paths: direct-light candidates, BSDF branches, visibility outcomes, shifted paths, caustic hints, and many small possibilities we will never explicitly enumerate.

The integrand f is the question the pixel asks: “How much radiance does this path explain?” The measure μ says where path mass lives. The image value is their pairing.

This is the first unification: light transport is not just path enumeration. It is evaluating observables against path-space measures.

A reservoir is a finite shadow of a continuous object

The continuous measure is too large. ReSTIR keeps a small weighted empirical measure, then repeatedly transports, reweights, and compresses it.

That means the selected sample is not the whole reservoir. It is merely the visible survivor of a larger weighted cloud. The weight sum, target density, proposal history, confidence, age, and transform provenance are part of the statistical object.

GRIS is transport between path-space domains

When a sample comes from a neighbor, a previous frame, an object reservoir, or a shifted path domain, it does not naturally live in the current pixel’s domain. A shift map brings it into the current fiber.

The Jacobian is not a mysterious correction factor. It is the density pressure created when transport stretches, compresses, folds, or splits the sample cloud.

Samples move forward; tests move backward

This is where the earlier pushforward/pullback essay plugs into the rendering equation. You can move the old reservoir forward into the current pixel and test it there. Or you can pull the current pixel’s test backward and evaluate the old reservoir in its native domain.

These are not two algorithms. They are one pairing, viewed from opposite sides.

Residual says what failed. Curvature says why reuse is unsafe.

Once transport is allowed, the renderer has a predicted measure for the current pixel. The incoming evidence is not just “the new frame”; it is a comparison target. Their difference is residual novelty.

But residual alone is not enough. A region can be noisy but stable. Curvature measures the more dangerous failure: two plausible reuse routes disagree.

The geometric machine

The whole series can now be compressed into one diagram. A pixel is a base point. Above it lives a path-space fiber. The reservoir is an empirical measure on that fiber. Reuse maps transport measures between fibers. Pullbacks transport tests in the reverse direction. GRIS weights correct density deformation. Residuals detect unexplained evidence. Curvature detects path-dependent reuse.

The central object is not “a sample.” It is a transported, tested, density-corrected empirical measure under finite memory.

The estimator, rewritten

In the usual form, we talk about sample candidates and weights. In the geometric form, we talk about where measures live, how they are transported, and whether the transported measure is still explanatory.

The implementation does not need to become abstract. It only needs to preserve the right diagnostic fields: source domain, transport map, Jacobian, confidence, residual, curvature, and novelty.

struct GeometricReservoir { Sample selected; float weightSum; float targetPdf; uint basePixel; uint sourceDomain; uint transportKind; float jacobian; float confidence; float residual; float curvature; float novelty; float entropy; }; GeometricReservoir update(Pixel p) { Measure pred = pushforward(previousReservoirs, transportsTo(p)); Evidence obs = estimateCurrentEvidence(p); float residual = distance(obs, pred); float curvature = commutatorDisagreement(p); float novelty = singularNoveltyProxy(obs, pred); CandidateSet C = proposeFrom( lights, bsdf, temporal, spatial, objectReservoirs, materialReservoirs); GeometricReservoir R; for (Candidate c : C) { Sample y = transport(c.sample, c.map); float J = estimateJacobian(c.map, c.sample); float target = evaluateTarget(p, y); float proposal = evaluateProposal(c); float trust = exp(-a*residual - b*curvature - c*novelty); updateReservoir(R, y, trust * target / max(proposal * J, EPS)); } return R; }

Compressed formulation: Light transport is the evolution, testing, compression, and correction of empirical measures over a screen-time bundle of path-space fibers. ReSTIR supplies the bounded empirical measure; GRIS supplies transport between domains; residuals and curvature decide when reuse should yield to new sampling.
